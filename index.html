解决Github Pages显示404

By Liu

发表于 2015-04-22

最近在某乎上查了下技术blog用Hexo弄比较方便，就自己弄下。我没有用域名，就先用Github Pages凑合着用。至于具体到步骤，已经有很多大神写了，请看参考文献，我就不再造轮子了。

经过一番配置，用Hexo在本地正常显示了，但是在浏览器打开http://runlittlecode.github.io还是显示404.看提示应该是在主目录缺少index.html文件，但是按大神说的步骤，运行

$hexo deploy

以后，应该就完事了。但运行完命令后，什么事情都没发生。github上面的项目还是没有index.html文件。看来问题就在这里了。

然后在网上各种找，发现都是点到为止，木有收获呀。最后还是翻官方文档吧，翻到deploy的时候，恍然大悟。

Before we start, you have to modify settings in _config.yml. A valid deployment setting must come with type field. For example:

deploy:
  type: git
 You can use multiple deployers. Hexo will execute each deployer in sequence.        
deployer:    
- type: git    
  repo:    
- type: heroku    
  repo

原来在deploy之前，需要在根目录的_config.yml文件中先设置发布的参数,也就是Deployment部分的参数(默认type是空的，所以我先前运行这个命令什么事情都没发生-.-||)。我是在Git上发布，所以按Git发布的流程来。

先安装hexo-deployer-git

$ npm install hexo-deployer-git --save

再编辑_config.yml的内容

deploy:
  type: git
  repo: git@github.com:RunLittleCode/RunLittleCode.github.io.git
  branch: master

配置完后，再运行发布的命令。这下有反应了，看了打印的log

$ hexo d
INFO  Deploying: git
INFO  Clearing .deploy folder...
INFO  Copying files from public folder...
On branch master
nothing to commit, working directory clean
To git@github.com:RunLittleCode/RunLittleCode.github.io.git
   c72f81b..ea7e6e7  master -> master
Branch master set up to track remote branch master from git@github.com:RunLittleCode/RunLittleCode.github.io.git.
INFO  Deploy done: git

好吧，原来是将public里面生成的静态文件上传到主目录了，这下主目录有index.html文件了。现在去打开http://runlittlecode.github.io,哦了,显示正常。

注意：在发布之前确认mac可以连到github,需要提前配置下ssh密钥,不然发布会有问题，具体怎么配，请看参考文献。
